Iframe playbook for the Atlas Coloring Book.

Canvas strips iframes from any domain its admin has not added to a trusted-iframe list. Pasting an iframe pointing at medmasterscollaborative.com into the Canvas HTML editor will silently disappear on save. Three ways around it, ranked by what you can do yourself.

Drop-in code

The iframe snippet

This is the iframe you would paste into Canvas's </> HTML editor. It points at the medmasterscollaborative URL. Replace the src value with your Canvas Files URL if you use Method B.

<!-- Atlas Coloring Book embed -->
<iframe
  src="https://www.medmasterscollaborative.com/atlas-coloring-book"
  title="Atlas Coloring Book"
  width="100%"
  height="1600"
  loading="lazy"
  allow="clipboard-write; fullscreen"
  style="border:1px solid #0B1530; border-radius:8px; max-width:1200px; width:100%;">
</iframe>
Why these attributes: title is required for screen-reader users (WCAG 4.1.2). loading="lazy" defers loading until the iframe scrolls into view. allow="clipboard-write; fullscreen" permits Save PNG and fullscreen drawing if you ever add it. No sandbox attribute, because the tool needs same-origin storage (IndexedDB) for the bank to persist between visits.

Method B · Recommended

Upload to Canvas Files, then iframe

Canvas always allows iframes that point at its own domain. So you put the HTML inside Canvas and iframe to it from the same Canvas page. No admin involvement needed.

  1. Open your BIO 004 Canvas course. Click Files in the left nav.
  2. Upload atlas-coloring-book.html into a folder of your choice. Make sure the file's visibility is Published.
  3. Right-click the file in the Files list and choose Copy link address. You will get a URL shaped like https://[your-school].instructure.com/courses/[course-id]/files/[file-id]/preview.
  4. Open the Canvas page where you want to embed. Switch to the HTML editor (the </> button at the bottom of the rich content editor).
  5. Paste the iframe snippet above, but replace the src value with the URL you copied. Save the page.
  6. Set a published students-can-access permission on the file. Files → right-click atlas-coloring-book.html → Manage Usage Rights → mark as Own creation → CC-BY or your preferred license, then Publish.
Note on the bank: When the tool is served from Canvas's domain, each student's saved bank persists in their browser's IndexedDB under your Canvas domain. Reminder for students to use Export book still applies.

Method A

Trusted-domain iframe (admin ticket)

If you want the iframe to point at the public medmasterscollaborative.com URL (so the same embed works for all your courses, and updates push automatically), Solano's Canvas admin has to add the domain to the Canvas Rich Content Editor's allowed-domains list.

  1. File a ticket with Solano IT or your Canvas administrator. Ask them to add medmasterscollaborative.com to the Rich Content Editor's allowed iframe sources at the institutional level (or at minimum, for BIO 004).
  2. Reason for the ticket: the domain hosts a CC BY-licensed anatomy practice tool you authored, used in BIO 004 lab prep. No PII flow, no tracking, no third-party cookies, IndexedDB-only local storage.
  3. Once approved, paste the iframe snippet exactly as written and save. Canvas will keep the iframe.
Heads up: Some Canvas instances strip iframes regardless and require an LTI 1.3 external tool registration instead. If admin says iframes are disabled categorically, fall back to Method B or Method C.

Method C

External URL module item (no iframe)

The lowest-friction option. You add the tool as a module item that either opens in a new tab or in a Canvas-framed iframe wrapper Canvas itself controls. No HTML editor work.

  1. Go to Modules. Pick the module you want the tool to live in (Week 1, Lab Prep, etc.).
  2. Click the + button on the module header and choose External URL.
  3. URL: https://www.medmasterscollaborative.com/atlas-coloring-book. Page name: Atlas Coloring Book.
  4. Check Load in a new tab (most reliable). If you leave it unchecked, Canvas will try to embed in its own iframe wrapper, which usually works for the tool but depends on the destination site's X-Frame-Options header.
  5. Add item and publish the module item.
Pairs nicely with your launcher page: the existing canvas-page-atlas-coloring-book.html works perfectly as a module page that explains the tool and links out. Method C is essentially what that page already does.

Bonus · If you want a stable hosting fallback

Serve from your GitHub Pages site too

You already have the repo at drsrennie-stack/atlas-coloring-book. Turn on GitHub Pages and the same file will be available at https://drsrennie-stack.github.io/atlas-coloring-book/atlas-coloring-book.html. Useful as a redundant URL if Kajabi goes down, and the file there picks up changes the moment you push.

  1. Go to the repo on GitHub. Click Settings → Pages in the left sidebar.
  2. Source: set to Deploy from a branch. Branch: main. Folder: / (root). Save.
  3. Wait 30–60 seconds. GitHub will give you a URL like https://drsrennie-stack.github.io/atlas-coloring-book/. The HTML file lives at that URL plus its filename.
  4. Update the iframe src (or your Canvas external URL) to the GitHub Pages URL if you prefer it as the canonical address.
